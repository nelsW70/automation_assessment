Summary

Tool

Selenium is a portable open source tool that provides a means to test web applications either by record/playback or scripting. As this tool was mainly designed to automate web applications, using it for a desktop/mainframe application would need third party software in addition. There would be risks to continuing on this path. Making updates to existing scripts would get more complicated. Our suggestion would be to consider using a commercial tool. Going this route will not only make scripting more efficient for such applications but also give more time/money towards increasing your test coverage.

SDLC Methodology

SDLC methodology and release cycle plan is important for the success of the product. From the survey completed by your organization, you did a very good job on selecting the proper release cycle with current SDLC methodology. Our SDLC team also has professional knowledge of SDLC methodology and release cycle plans. We can work with your team, adding automation execution into your current release process to implement Continuous Integration/Continuous Development.

Test Coverage

Test coverage helps you identify how thoroughly testing has been performed. Number of test resources are dependent on the complexity of the code. As per your submitted survey, we see that test coverage is low with more resources. One of the most likely reasons is that tests are not being automated and resources end up doing more manual testing. We as a team at SDLC can help you achieve better test coverage by providing the best automation practices and thus delivering a defect-free product.

Defect Detection

It seems like your current percentage of production defects is standard. Defect detection is extremely critical in process automation, especially in cases with high coverage. In this case, having a standard defects detection is manageable but there will be cases where defects would have to be tracked down manually which costs time. SDLC will provide you with resources to optimize the script.
